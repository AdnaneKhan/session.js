## What

`GroupManager` inbound NEW handling (`#handleNew`, plan §2.2 gates), wired via
the client's mapped `groupUpdate` event:

1. sender approved **or self** (`isSenderApproved` option, default accept-all —
   a programmatic agent sets its own policy; self always allowed);
2. we are in `members`;
3. `name`/`publicKey`/`members`/`admins`/`encryptionKeyPair` all present;
4. `publicKey` is a **legacy (05) address, not v3 (03)**;
5. **dedupe** — if the group is already known & active, just append the keypair
   (dedupe-by-value), no re-join.

On join: store group (`lastJoinedTimestamp = envelope.timestamp` watermark),
append the keypair, start polling, emit `groupJoined`. Two `GroupManager`s over
one `GroupBus` exercise the full creator→invitee flow offline.
