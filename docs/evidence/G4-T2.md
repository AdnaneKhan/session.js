## What

`GroupManager.createGroup({ name, members, expirationTimer? })` (plan §2.2):

- Validates name + member Session IDs; dedupes members and always includes
  ourselves; enforces the 100-member limit (`GroupTooLargeError`).
- Generates the group address + first encryption keypair (G4-T1); stores the
  group state (`admins = [us]`, `lastJoined/formation = now`) and appends the
  keypair.
- Sends **one NEW DM per member (including self)** via
  `sendClosedGroupUpdate` (DM mode → ns 0, sealed to each member's identity
  key; the group keypair travels plaintext inside the sealed box).
- Starts polling the group swarm (`addGroupPoller`) and emits `groupCreated`.
- Offline harness: `test/helpers/bus.ts` — a mapped-event-level `GroupBus` +
  `BusGroupSession` (the calls `SignalingBus`/`BusSession` pattern) relaying
  sends to member endpoints like a swarm poll delivers.
