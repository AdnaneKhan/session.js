## Summary

New `sendCallMessage(this: Session, to, callMessage, options?: { isSyncMessage?: boolean })`
in `src/instance/send-call-message.ts`, mirroring `src/instance/typing-indicator.ts` shape
and `sendMessage`'s guards:

- Guards: `SessionRuntimeError(EmptyUser)` if keys/sessionID unset; `to` must be 66-char hex
  starting "05" (`SessionValidationError(InvalidSessionID)`) — same validation as sendMessage.
- Flow: `timestamp = getNowWithNetworkOffset()` → schema `CallMessage` → `toRawMessage(to, msg,
  SnodeNamespaces.UserMessages)` → `wrap(keys, [{destination: to, plainTextBuffer, namespace, ttl,
  identifier, isSyncMessage: options?.isSyncMessage ?? false, isGroup: false}], {networkTimestamp:
  timestamp})` → `_storeMessage` → returns `{ messageHash, timestamp }`.
- Bound as `public sendCallMessage = sendCallMessage.bind(this)` in `src/instance/index.ts`
  with TSDoc documenting self-sync (isSyncMessage=true stores to own swarm for ANSWER/END_CALL
  so linked devices stop ringing; destination must be own Session ID; calls package invokes
  twice: once to peer, once to self).

wrap() pass-through verified: `overwriteOutgoingTimestampWithNetworkTimestamp` in
`src/crypto/message-encrypt.ts` only rewrites dataMessage/dataExtractionNotification/typingMessage
timestamps; CallMessage has no timestamp field and re-encodes byte-identically.

Provenance: written fresh from the published SessionProtos.proto field facts. No lines
copied from GPL/AGPL sources. Licensable under MIT for upstream contribution.
