## What

`@session.js/groups` foundations (AGPL):

- **`GroupSessionLike`** (`src/types.ts`): the structural interface the package
  needs from a patched client Session — `getSessionID`, `getNowWithNetworkOffset`,
  `on/off` for the mapped `groupUpdate` / `message` / `syncClosedGroups` events,
  `sendClosedGroupUpdate`, `sendGroupMessage`, and `addGroupPoller` /
  `removeGroupPoller`. Method-style declarations (checked bivariantly) so a real
  Session satisfies it with at most one boundary cast — the @session.js/calls
  doctrine. Also the plain-number `GroupControlMessageType` enum bridge, event /
  state / keypair types, and `GroupManagerEventMap`. Builds WITHOUT the client
  installed (no protobuf bindings imported — the client delivers mapped events).
- **`GroupManager`** (`src/group-manager.ts`, lifecycle wiring adapted from
  desktop `closed-group.ts` / android `ClosedGroup.kt`): `extends EventEmitter`,
  constructor `(session, options?, deps?{ storage })`, subscribes to the three
  session events with bound listeners, `init()` loads known groups, typed
  `on/off/once/emit` overloads, `#guard` containment, a throwing-consumer-logger
  guard, idempotent `dispose()` (unsubscribes + stops group pollers), and
  `saveGroup` / keypair accessors. Inbound handler bodies are filled in by
  P4–P7 on this scaffold.
- **Error taxonomy** (`src/errors.ts`), **barrel** (`src/index.ts`).
