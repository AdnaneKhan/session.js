## What

Core (MIT) additions for the legacy closed-group control message:

- `src/messages/schema/closed-group-control-message.ts` — `ClosedGroupControlMessage`
  schema class (extends `ContentMessage`), all six sendable control types
  (NEW / ENCRYPTION_KEY_PAIR / NAME_CHANGE / MEMBERS_ADDED / MEMBERS_REMOVED /
  MEMBER_LEFT; `ENCRYPTION_KEY_PAIR_REQUEST` rejected as unused), per-type
  validation (admins ⊆ members, keypair required for NEW, etc.), 14-day content
  TTL, `contentProto()` emitting `Content({ dataMessage: { closedGroupControlMessage } })`
  — **no `GroupContext`** on control messages (verified against pinned
  session-desktop `ClosedGroupMessage`/`ClosedGroupNewMessage`/… dataProto).
- `src/messages/index.ts` — `ClosedGroupUpdate` mapped type +
  `mapClosedGroupControlMessage` mapper (group-swarm vs 1:1-DM envelope
  semantics: `groupId` from explicit `publicKey` else `envelope.source`; `from`
  from `senderIdentity` for group messages else `envelope.source`).
- `src/instance/events.ts` — `groupUpdate: (update: ClosedGroupUpdate) => void`.
- `src/instance/polling.ts` — dispatch `groupUpdate`, and exclude control
  messages from the visible-`message` chain.
- Exported from `src/messages/schema/index.ts`.

Written fresh from the published `SessionProtos.proto` field facts and the
pinned reference shapes — MIT-licensable, upstreamable.
