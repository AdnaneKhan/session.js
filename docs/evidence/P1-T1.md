## Summary

`mapCallMessage` in `src/messages/index.ts` previously dropped `sdps`, `sdpMLineIndexes`,
`sdpMids` and did not surface the envelope timestamp. The `CallMessage` event payload type
was extended with `timestamp`, `sdps`, `sdpMLineIndexes`, `sdpMids` (default `[]`) and the
mapper now copies them with `[...(c.sdps ?? [])]`-style spreads. `from: envelope.source` is
preserved (same sender resolution as all other private-message mappers — Q4). Envelope
timestamp uses the same `typeof timestamp !== "number" → toNumber()` Long handling as
`mapDataMessage`.

Provenance: written fresh from the published SessionProtos.proto field facts. No lines
copied from GPL/AGPL sources. Licensable under MIT for upstream contribution.
