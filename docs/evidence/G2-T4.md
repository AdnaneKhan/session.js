# G2-T4 — `GroupContext` in visible sends + legacy config `activeClosedGroups` parse/emit

- **Status:** done
- **Agent:** main (implementer)
- **Recorded:** 2026-07-21T12:54:08Z

## What

- **`GroupContext` in visible sends** — `VisibleMessage` gained an optional
  `groupContext: { groupId }` param; `dataProto()` now emits
  `DataMessage.group = GroupContext{ id: utf8(groupId), type: DELIVER }`
  (verified against pinned session-desktop `ClosedGroupVisibleMessage`, which
  utf8-encodes the 05-prefixed hex and sets DELIVER).
- **Legacy config `activeClosedGroups` parse** — new `ClosedGroupConfig` type +
  `mapConfigurationClosedGroups(content)` mapper in `src/messages/index.ts`
  (extracts `{ publicKey, name, encryptionKeyPair{publicKey,privateKey},
  members, admins }` from `ConfigurationMessage.closedGroups`, skipping entries
  missing a keypair). Emit already existed (`ConfigurationMessageClosedGroup` /
  `ConfigurationMessage.activeClosedGroups`).
- **`syncClosedGroups` event** — added to `EventCallbackSignatures`; the poll
  dispatch now parses closed groups out of the latest inbound
  `ConfigurationMessage` and emits `syncClosedGroups` (reconciliation lives in
  the groups package — P7).
