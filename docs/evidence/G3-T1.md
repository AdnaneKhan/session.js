## What

`@session.js/groups` foundations (AGPL):

- **Storage schema** (`src/storage.ts`): `StorageLike` structural interface;
  `GroupStorage` typed accessors over the dynamic-prefix key convention —
  `closed_group_index`, `closed_group:{id}:state|:keypairs|:last_hashes|:undecryptable`;
  `deleteGroup` wipes all package-store keys + index entry and tolerates corrupt
  JSON. The core poller's identically-prefixed cursor/retry keys live in the
  Session storage and are explicitly cleared through `clearGroupPollerState`.
  `InMemoryGroupStorage` default.
- **Keypair registry** (`src/keypairs.ts`, ported from desktop's
  `getAllCachedECKeyPair` / `addKeyPairToCacheAndDBIfNeeded` /
  `removeAllClosedGroupEncryptionKeyPairs`): in-memory cache + storage
  persistence; **append order** (newest last), **dedupe by value**
  (public+private), `getLatest` = last appended, `removeAll`. Semantics per
  IMPLEMENTATION.md §2.5 (all historical pairs retained; no timestamp ordering).
