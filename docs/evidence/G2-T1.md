## What

MIT core patches:

- **Public `./crypto` export** (`src/crypto/index.ts` + `package.json`
  `"./crypto": "./dist/crypto/index.js"`): re-exports `encrypt`, `wrap`,
  `encryptUsingSessionProtocol`, `decryptMessage`, `decodeMessage`,
  `extractContent`, `decryptEnvelopeWithOurKey`, `decryptForClosedGroup`,
  `decryptWithSessionProtocol`, padding helpers, `cryptoBoxSeal/Open` + the
  box-size constants, and `getSnodeSignatureParams`. Previously internal-only.
- **`encrypt()` closed-group branch implemented** (was dead commented code):
  for `CLOSED_GROUP_MESSAGE`, seals to the group encryption key via the Session
  protocol and returns `envelopeType: CLOSED_GROUP_MESSAGE`. `wrap()` gained an
  optional `encryptionPublicKey` field so a group message is sealed to the group
  encryption key while the envelope `source` stays the group address.
- **`decryptForClosedGroup` fixed** — two upstream bugs:
  1. the successful-decrypt result was shadowed by a loop-scoped `const`, so
     every closed-group decryption failed unconditionally;
  2. `.pop()` drained the caller's keypair array.
  Now iterates newest-first non-destructively and unpads on success.
- **`encryptUsingSessionProtocol` exported** so the groups package can build
  keypair wrappers (Session-protocol seal, NO message padding — verified against
  pinned session-desktop `buildEncryptionKeyPairWrappers`, which does
  `encryptUsingSessionProtocol(memberPubKey, KeyPair-proto-bytes)`).
