## What

Public `Session` methods (`src/instance/send-group-message.ts`, bound in
`src/instance/index.ts`; `sendCallMessage` precedent):

- **`sendGroupMessage({ to, encryptionPublicKey, text, timestamp?, expirationType?, expireTimer? })`**
  — visible group chat message. Builds a `VisibleMessage` with a `GroupContext`
  (`id = utf8(to)`, `DELIVER` — see G2-T4), seals to the group encryption key via
  `wrap(..., isGroup: true, encryptionPublicKey)`, envelope
  `CLOSED_GROUP_MESSAGE` (source = group address), stored to the group swarm at
  namespace −10.
- **`sendClosedGroupUpdate({ to, controlMessage, encryptionPublicKey?, timestamp? })`**
  — sends a `ClosedGroupControlMessage` (P1 schema). Routing by presence of
  `encryptionPublicKey`:
  - **group mode** (provided): sealed to the group encryption key,
    `CLOSED_GROUP_MESSAGE` envelope, namespace −10 (NAME_CHANGE / MEMBERS_* /
    MEMBER_LEFT / rotation ENCRYPTION_KEY_PAIR);
  - **DM mode** (omitted): sealed to the member identity key `to`,
    `SESSION_MESSAGE` envelope, namespace 0 (NEW invites, ENCRYPTION_KEY_PAIR
    replies carrying an explicit `publicKey`).

Both validate the initialized instance, the 05-prefixed 66-char destination, and
the unprefixed 64-char encryption key.
