## What

Receive path via the core `GroupPoller` (built G2-T2, extended here):

- Decrypts each retrieved group-swarm message with the injected keypair
  registry, **newest-first** (the fixed `decryptForClosedGroup`), falling back
  through **historical (rotated-out) keypairs** so in-flight / pre-rotation
  messages still decrypt.
- Recovers the real author into `envelope.senderIdentity`; hash-based per-device
  deduplication suppresses the sender's network echo without suppressing linked
  devices that share the same Session ID.
- Refactored into a guarded `#tryDecrypt` result shared by the fresh-message
  path and the retry path (G5-T3), including malformed-envelope containment.

The groups `GroupManager` supplies the keypairs via
`getEncryptionKeyPairs`/`getLatestEncryptionKeyPair` and consumes the mapped
events (G5-T1).
