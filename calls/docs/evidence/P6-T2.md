## Model

`SharedFakeSession`: ONE SessionLike for account B whose "call" listener list
fans every emitted message to BOTH CallManagers B1 and B2 (same getSessionID
— two linked devices polling the same swarm). Self-sync sends
(`isSyncMessage` / to own id) are fanned back into the listener list exactly
as the own-swarm poll would deliver them to the other device. Offline; real
supervisor logic on both sides.

## Verification

```
cd calls && bun test test/multi-device.test.ts    # exit 0 — 3 pass
```

- Caller A sends PRE_OFFER+OFFER → **both** B1 and B2 emit incoming
  (remote-ring).
- B1.accept() → exactly two wire messages: ANSWER(peer A) + ANSWER(self).
  The self-ANSWER fans into B2 → B2 ends **`answered-elsewhere`**, media
  closed — and B2 sent **zero** wire messages after being silenced (post
  accept, the send log contains only B1's two messages).
- B1.reject() → END_CALL peer + END_CALL self → B2 ends `ended-elsewhere`,
  likewise silent.
- Self-sent PRE_OFFER/OFFER/ICE_CANDIDATES/PROVISIONAL_ANSWER from the own
  swarm (from == own id) are dropped by BOTH devices: no incoming, no ended,
  no wire output, no errors, no crash.
- B1's own receipt of its self-ANSWER is a no-op (self-answer invalid in
  `connecting` — logged + swallowed; the call continues).

`answered-elsewhere` / `ended-elsewhere` EndReasons are in the normative
type surface (types.ts) and were already exercised at supervisor level; this
task proves them at the PUBLIC two-manager topology.
