## Deliverables

- `examples/voice-agent.ts` — `startVoiceAgent(session, opts)` + exported
  `STTEngine` / `TTSEngine` interfaces, offline defaults `EchoStub` (buffers
  PCM, replies after 200 ms) + `PassthroughTTS`. Inbound mode auto-accepts
  (on remote-ring — accepting on PRE_OFFER races the OFFER; documented),
  outbound mode (`--call 05…`) writes 3 s of 440 Hz sine and records received
  PCM to `examples/out.pcm`. Session bootstrap via dynamic imports (the
  PATCHED fork must satisfy SessionLike at runtime; zero compile-time
  dependency).
- `examples/offline-echo.ts` — two CallManagers over an in-process signaling
  bus with the REAL werift media plane: A calls B, B's echo pipeline returns
  the audio; asserts frames both directions.
- `examples/README.md` — quickstart + how to plug Whisper.cpp (spawn
  `whisper-cli -s`, resample 48k→16k) and any TTS (Piper/espeak → resample
  to 48k/960-sample frames), audio contract, v1 limitations.

## Verification run

```
cd calls && bun examples/offline-echo.ts     # exit 0
```

```
[B:info] incoming call from 05aaaa…

offline-echo: PASS (pipeline round-trips real Opus audio)
```

100/100 frames received by B; 99 echo frames returned to A (≤1 frame startup
loss — within the ≤2 % documented loopback budget). Exercises the full stack:
CallManager → CallSupervisor FSM → SessionSignaling/TrickleIceSender over the
bus → werift ICE/DTLS-SRTP → AudioBridge Opus enc/dec → STT/TTS pipeline →
writeAudio.

## Bugs found & fixed by this demo

1. Auto-accept on `incoming` rejected with "accept before OFFER received"
   (PRE_OFFER rings first) — accept now deferred to `remote-ring`.
2. werift early-media-connect race (supervisor deferral fix — see P4-T3.md).
3. Bilateral approval required (B must `approveContact(A)` — contacts-only
   gate is symmetric; README documents).
