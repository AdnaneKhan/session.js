## werift TURN findings (binding for P4-T1)

1. **URLs MUST carry an explicit port.** `parseIceServers` (webrtc
   `utils.js`) does `url.split(":")` → `[address, parseInt(port)]`;
   `"turn:host"` (Session's constants have no port) yields port `NaN` and
   the allocation fails SILENTLY (error only in `debug` logs). The
   PeerConnectionManager normalizes ICE server URLs to append `:3478` when
   no port is present.
2. **Only the FIRST `turn:` iceServer is used** (`.find()`); werift has a
   single `turnServer` slot. The shuffle-take-2 selection (policy.ts) still
   matters operationally (which 2 we pass → the first wins), and we pass
   both for config parity, but failover across servers is NOT automatic in
   werift. Documented as a known limitation.
3. TURN long-term credentials (username/password) work correctly with the
   official servers (401 → nonce retry → Allocate). werift also falls back
   UDP→TCP transport automatically (`turnTransport` handling).
4. `iceTransportPolicy: "relay"` maps to `forceTurn: true`
   (`secureTransportManager.js`).
