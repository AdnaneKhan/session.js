## Deliverable

`calls/e2e/harness.ts` + `calls/e2e/README.md` — the framework behind the
Tier-1/Tier-2 CLIs:

- **Two backends**
  - OFFLINE: `SignalingBus` + `BusSession` (in-process swarm, the
    examples/offline-echo.ts pattern, extended with a wire SPY,
    per-endpoint send accounting, delivery-delay injection, and a dropper
    hook) + the REAL werift `PeerConnectionManager` media plane with
    `iceServers: []` (host-candidate ICE, real UDP + DTLS-SRTP, ZERO TURN
    traffic — plan R7).
  - NETWORKED (gated `SESSION_CALLS_NETWORK_TESTS=1`): real patched-client
    `Session` instances with freshly generated accounts
    (`@session.js/mnemonic` `encode(generateSeedHex())`), Poller 500 ms,
    fork src imported structurally (`session as never` — the same bridge
    the networked integration test uses).
- **Deterministic audio:** `SineSource` (default 440 Hz, 48 kHz mono
  Int16, 960-sample frames, via `sineFrame`) + `ToneDetector` (Goertzel
  `toneSnrDb`, 20 dB threshold vs the spike's ~198 dB clean-path SNR).
- **Timing collector:** ring / connect / audio-first-frame / total per
  scenario (`TimingCollector`).
- **Evidence writer:** `writeRunReport` → `e2e/reports/<run-id>/report.json`
  + optional raw `.pcm` captures. Mandatory sanitization pipeline:
  `redactSensitive` → caller-secret scrubbing (generated mnemonics) →
  `FORBIDDEN_REPORT_MARKERS` scan (TURN creds, DTLS fingerprints, SDP
  bodies `v=0`/`o=-`/`m=audio`, PEM) → JSON re-parse — asserted clean
  BEFORE the file touches disk.
- **Plumbing:** `Checks` accumulator, per-scenario guard timeouts
  (`withGuard`), unhandled-rejection tracker, table printer,
  `verifyWireFixtures` shell-out.
