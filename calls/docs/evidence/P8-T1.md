## Deliverable

`calls/README.md` — full documentation replacing the quickstart stub (the
stub's working quickstart snippet is retained verbatim in §3):

1. **Quickstart** — install (tarballs / fork checkout; the published
   0.0.57 client lacks the patch), register/load an account
   (`encode(generateSeedHex())` + `Poller`), place/answer calls with PCM
   hooks, runnable examples incl. `bun run example:echo` (offline
   full-stack), and the E2E suite commands.
2. **API reference** — generated from the actual TSDoc surface in
   `src/`: `CallManager` + `CallManagerOptions` (all 8 options with
   defaults) incl. the documented additive `approveContact` /
   `isContactApproved`; `Call` surface; `CallState` / `EndReason` (incl.
   the multi-device reasons); manager + call events; the audio contract
   (48 kHz / mono / Int16 / 960-sample frames, no internal pacing,
   backpressure semantics); error taxonomy; the `SessionLike` structural
   contract for custom adapters (with the wire enum values); advanced
   `MediaEngine` / `PeerConnectionManager` / `AudioBridge` wiring.
3. **Architecture overview** — the plan §4.2 ASCII diagram adapted to the
   real components: CallManager → CallSupervisor (FSM, gates, timers) +
   SessionSignaling (peer/self-sync) + TrickleIceSender (200 ms batch,
   relay filter) → PeerConnectionManager / AudioBridge → werift
   RTCPeerConnection (ICE/DTLS-SRTP, negotiated "signaling" channel id
   548, Android-parity munging) over the forked client's
   `sendCallMessage` / `call` event / `setPollInterval`.
4. **Limitations & caveats** — audio only; one concurrent call;
   contacts-only gating + `approveContact` workflow; callee beta opt-in
   → silent 60 s timeout; **relay-only broken in v1** (P5-T2 link) with
   the P2P-first default working; polling-latency expectations (500 ms
   boosted / 3000 ms default, measured numbers); Node ≥ 22 primary, Bun
   secondary; no onion routing.
5. **TURN ethics note** (plan §3.4 / R7) — nonprofit infrastructure,
   self-host coturn with the same static credentials for fleets, use
   relay sparingly, zero TURN traffic in the offline suite.
6. **Privacy disclosure** (R5) — no onion routing; caller IP visible to
   snodes; callee IP visible in P2P; relay would hide IPs but is broken
   in v1; "do not market as anonymity-preserving".
7. **License section** — AGPL-3.0-or-later, `COPYING.provenance`
   (per-file porting provenance), and the §13 network-use obligation
   stated plainly (complete corresponding source for network services).
