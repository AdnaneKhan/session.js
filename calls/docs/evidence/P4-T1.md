## Test coverage vs task spec

1. **SDP munging units** (7 tests): `;cbr=1` appended exactly once to the
   opus fmtp of a realistic multi-codec SDP (111 opus + PCMU/PCMA/G722);
   dynamic PT discovery (tested with PT 96); fmtp line CREATED when opus
   has none (werift's default — empty codec parameters → no fmtp);
   `ssrc-audio-level` extmap removed, other extmaps untouched; credentials/
   fingerprint/BUNDLE untouched; idempotent (`munge(munge(x)) === munge(x)`);
   CRLF and LF preserved.
2. **Loopback negotiation** (2 tests): two managers (caller/callee) reach
   the mapped `"connected"` state over host candidates well inside the 15 s
   budget (~0.5–0.8 s observed); offer SDP verified munged (cbr=1 present,
   ssrc-audio-level absent); state mapping dedupes and applies hysteresis
   (see finding 2); late `onConnectionState` subscribers replay current state.
3. **Data channel roundtrip** (2 tests): `{"hangup":true}` observed by the
   peer via `onDataChannelMessage`; `{"video":true}` fires
   `onRemoteVideoToggle(true)`; negotiated channel id 548 / label
   `signaling` opens on both roles.
4. **ICE restart** (1 test): `createOffer({iceRestart:true})` yields a
   different `a=ice-ufrag` than the previous local description; munging
   still applied to the restarted offer.
5. **Leak sanity** (2 tests): 20 full create/connect/close cycles — zero
   unhandled rejections (process-level guard), idempotent `close()`;
   plus 100 rapid create/close cycles without connecting — zero rejections.
6. **Garbage candidates** (2 tests): `addRemoteCandidate` with junk, empty,
   and invalid-IP candidate strings resolves without throwing and does not
   disturb an established session; candidates received before the remote
   description are buffered (werift throws otherwise) and drained after.

## Implementation notes / findings (Wave-3 intel)

1. **werift config mapping** (`toWeriftIceServers` + PeerConfig):
   `iceServers` flattened to single-string urls with explicit `:3478`
   default ports (P3-T2 finding — werift parseInts a missing port to NaN
   and fails silently); `iceTransportPolicy` honored via `forceTurn`;
   `bundlePolicy: "max-bundle"`. **Gaps**: no `rtcpMuxPolicy` key (werift
   always rtcp-muxes ≡ `require`) and no `sdpSemantics` (Unified Plan only).
   Both gaps documented in the file header.
2. **ICE state flap → hysteresis**: ~25 ms after nomination werift briefly
   re-emits `checking` while remaining pair checks run
   (`connected → checking → connected` observed). The mapper suppresses a
   `connected → connecting` downgrade (RFC 8445 connected-stickiness); after
   a real `disconnected`, `connecting` flows again for ICE-restart flows.
3. **Local SDP is the MUNGED SDP**: `createOffer`/`createAnswer` set the
   munged string as the local description (matches session-desktop
   behavior); werift parses `cbr=1` as an opaque string fmtp parameter —
   no internal-state mismatch observed (all negotiations succeed).
4. **Remote candidates are gated**: buffered until the remote description
   is set (werift's `addIceCandidate` throws before then) — mirrors the
   supervisor's `readyForIce` rule (plan §3.2).
5. **Teardown**: `close()` is synchronous (contract), idempotent,
   unsubscribes every werift `Event` subscription, closes the data channel,
   stops the local track, and fire-and-forgets `pc.close()` with rejections
   contained (P6-T3: media errors must never crash the host).
6. **Batching hook point for Wave-3**: local candidates surface via
   `onLocalCandidate(cb)` one at a time (extracted `{candidate,
   sdpMLineIndex, sdpMid}` triple); the supervisor's existing
   `IceCandidateBatcher` (policy.ts, 200 ms debounce) plugs directly onto
   it. Note: relay policy leaks host/srflx candidates into the callback
   (werift gathers all types but pairs on relay only) — Wave-3 may filter
   `typ host|srflx` before sending when `iceTransportPolicy === "relay"`.
