  (non-initiator entry into `reconnecting`; guard non-initiator). Matrix now
  20 events / 22 rows (test/fsm/matrix.test.ts updated — all behavioral

- Initiator (outbound): on media "disconnected" from connected → ice-disconnected
  → pending-reconnect → restart loop: OFFER(iceRestart, same uuid, fresh SDP)
  every `ICE_RESTART_INTERVAL_MS` (5 s), ≤ `ICE_RESTART_MAX_ATTEMPTS` (5).
  FSM restart-attempt → reconnecting (first attempt; later attempts stay
  reconnecting and resend). Restart ANSWER → receive-answer (reconnecting →
  connecting) → setRemoteAnswer → ice-connected restores connected (attempts
  counter + timer cleared). Exhaustion → IceFailureError on the error channel
  + END_CALL peer+self + endReason `ice-failed`.
- Non-initiator (inbound): network-reconnect → reconnecting + wait
  `NON_INITIATOR_RECONNECT_WAIT_MS` (60 s, policy.ts — spec §3.2 "waits
  ≤60 s"). Restarted OFFER → receive-offer-restart → ANSWER peer+self
  (answer-icerestart). Elapse → IceFailureError + END_CALL + `ice-failed`.

## Timer strategy (documented choice)

Bun has no vitest-style fake-timer API; the cadence constants are fixed
policy. The supervisor takes an injectable timer pair
(`CallSupervisorDeps.timers`), threaded through a NON-normative additive
third CallManager constructor argument (`CallManagerDeps.timers`, also media
override for fakes). The normative consumer signature
`new CallManager(session, options?)` is unchanged.
