## What is logged (options.logger; all strings through redactSensitive)

- **Every FSM transition** — `fsm: <from> --<event>--> <to>` (StateMachine's
  injected logger; covers all transitions incl. reconnect rows).
- **Every signaling send/recv** — `signaling send peer|self type=<NAME> uuid=…`
  (SessionSignaling) and `signaling recv type=<NAME> uuid=…` (supervisor
  trace). Direction/type/uuid ONLY — SDP payloads are never passed.
- **Call summary on end** — `call ended uuid=… peer=… direction=… endReason=…
  durationMs=… connectedMs=… icePairType=unknown framesDelivered=…
  framesDropped=…` (+ structured `meta.summary`). framesDelivered/
  framesDropped come from the AudioBridge counters when the session is a
  WeriftMediaSession (else 0). `icePairType` is always `"unknown"` in v1 —
  the MediaSession binding does not expose the selected candidate pair
  (documented media-layer delta for a future wave).
- **Redaction** — the CallManager wraps the consumer logger: message strings
  AND the JSON snapshot of meta pass through `redactSensitive` (TURN
  username/password anywhere; DTLS fingerprints — strengthened in P5-T3 to
  match mid-line too, e.g. embedded in error messages). Errors in meta are
  serialized as {name, message, code} then redacted.
